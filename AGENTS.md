# AGENTS.md

Guidelines for AI coding agents working in this repository.

## Project Overview

This is a **static personal portfolio website** for Vuong Hoang Thu Truc.
The site is generated by **Framer** (a no-code web design tool) and deployed
via **GitHub Pages** at the custom domain `trucvuong.site`.

### Repository Structure

```
/
├── AGENTS.md        # This file — agent guidelines
├── CNAME            # GitHub Pages custom domain config (trucvuong.site)
├── Plan.md          # Brief planning note (Vietnamese)
└── index.html       # The entire website (~7,400 lines, single-page)
```

There is no `src/` directory, no components, no modules. Everything lives in
`index.html`.

## Build / Lint / Test Commands

**There are none.** This project has:

- No package manager (no `package.json`, no `node_modules/`)
- No build tool (no Vite, Webpack, Rollup, etc.)
- No linter (no ESLint, Stylelint, etc.)
- No formatter (no Prettier, etc.)
- No test framework (no Jest, Vitest, Playwright, etc.)
- No CI/CD workflow files

### Deployment

Deployment is automatic via **GitHub Pages**. Pushing to the `main` branch
publishes the site. The `CNAME` file maps the repo to `trucvuong.site`.

### Local Preview

To preview locally, open `index.html` in a browser or run a simple HTTP server:

```sh
# Python 3
python3 -m http.server 8000

# Node.js (if npx available)
npx serve .
```

Then visit `http://localhost:8000`.

## Code Style Guidelines

### General Architecture

The `index.html` file is a **Framer-generated export**. It contains:

1. **Inline CSS** — All styles are embedded in `<style>` tags with Framer
   class naming conventions (e.g., `.framer-9LwVP`, `.framer-RTHx6`).
2. **Inline JavaScript** — Framer runtime scripts and scroll animation code
   are embedded in `<script>` tags.
3. **External CDN references** — Fonts from Google Fonts, assets from
   `framerusercontent.com`, and the Framer runtime from `framer.com`.

### HTML Conventions

- The document uses `lang="en"` and UTF-8 encoding.
- Meta tags include Open Graph and Twitter Card metadata for social sharing.
- The page uses responsive breakpoints at 810px and 1199px via CSS media
  queries for mobile, tablet, and desktop layouts.
- Framer uses `data-framer-*` attributes extensively for component binding.

### CSS Conventions

- **Class names**: Auto-generated by Framer using the pattern `framer-XXXXX`
  (e.g., `framer-1gts3xw`). Do not rename these.
- **CSS custom properties**: Used for theming via `--token-*` variables
  (e.g., `--token-7e7cbfd8-1564-4e68-b512-030af686b3b7`).
- **Responsive design**: Uses `@media` queries with min/max-width breakpoints.
- **Fonts**: DM Serif Display, Fraunces, and Inter are loaded from Google Fonts.
- **Layout**: Flexbox-based layout throughout.

### JavaScript Conventions

- Custom scroll animations were added manually (see git history: commit
  `892ce0b`), using `IntersectionObserver` for fade-in-up and stagger effects.
- The Framer runtime is loaded from CDN — do not modify or replace it.
- Any custom JS should be added in `<script>` tags at the end of `<body>`.

### Naming Conventions

- **Framer-generated names**: Leave auto-generated class names and IDs as-is.
  They are opaque identifiers (e.g., `framer-f6SdY`).
- **Custom additions**: When adding custom CSS classes or JS, use descriptive
  kebab-case names (e.g., `scroll-animate`, `fade-in-up`) to distinguish
  them from generated code.

### Error Handling

- There is no application-level error handling — this is a static site.
- If adding custom JavaScript, use `try/catch` blocks and fail gracefully
  (the site should always render even if JS fails).

## Important Warnings

### Do Not Regenerate from Framer

The HTML was exported from Framer and then **manually modified** (scroll
animations, meta tags, favicon removal). Re-exporting from Framer would
overwrite these customizations.

### Editing the index.html

- The file is ~7,400 lines. Be precise with edits — use targeted search
  and replace rather than rewriting large sections.
- Preserve Framer's generated structure. Only add or modify code in clearly
  separated sections.
- When adding new sections, add clear HTML comments to mark custom code:
  ```html
  <!-- Custom: Section Name Start -->
  ...
  <!-- Custom: Section Name End -->
  ```

### Assets

- Images and media are hosted on `framerusercontent.com` CDN. They are not
  stored in this repository.
- If adding local assets, create an `assets/` directory at the repo root.

### Git Practices

- Keep commits focused and descriptive.
- The commit history shows the convention: short imperative messages
  describing the change (e.g., "Add scroll animations: fade-in-up, stagger
  effects on all sections").
- Do not commit large binary files — use external hosting for media.
